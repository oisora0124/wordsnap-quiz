# WordBank 継続レビュー記録（2026-07-21）

## 結論

既存のJSON、個人リンク、旧D1行、学習履歴を維持したまま適用できる防御と回帰検査は実装済み。
認証方式、サーバー履歴、復習アルゴリズム、語義モデルの本番接続は影響が大きいため、
移行・復旧・影運用の受入条件が揃うまで変更していない。

レビューは動的オーケストレーション方針のモードEを優先し、Codexだけで実施した。
決定論的検査に合格した低リスク変更だけをmainへpushした。

## 実施した監査と改善

### データ保護・同期

- 破損したD1状態を空状態として返さず、`corrupt_state`で自動処理を停止する。
- 旧プレーン行、任意形式の旧キー、学習スキーマv0を読み続けられることを契約検査する。
- 競合時の学習状態を保守的に統合し、未来へずれた端末時刻を優先しない。
- 削除墓標が削除語の復活を防ぎ、別デッキの同名語を巻き込まず、再追加を許すことを
  実コードで回帰検査する。
- 1万語のgzip保存・読込・競合をD1互換テストで往復する。
- 同期通信と保存版生成に60秒の上限を設け、無期限待機を防ぐ。
- GET／PUT／409応答のキー、revision、状態形状を検証し、曖昧な成功応答では自動処理を止める。
- 同一キーへの同時初回保存・同一revision更新が1件だけ成功し、敗者へ勝者の完全状態を返すことを動的検査する。
- キー切替後に遅着した旧キー応答を、通信中断が効かない場合も世代番号で破棄することを動的検査する。
- PUT待機中のローカル変更が、完了後の単一キューでちょうど1回再送されることを動的検査する。
- JSON読込、全削除、強制取得の前に端末内Undoを作る。

### 秘密情報・プライバシー

- 新規個人キーはWeb Cryptoの240ビット乱数だけで生成し、利用不能時は安全に停止する。
- `meta referrer=no-referrer`とCloudflare Pages用`_headers`の両方で、個人キー入りURLの
  Referer送信を抑止する。
- リポジトリ全体のテキストを対象に、APIキー・トークン・秘密鍵らしい値を公開前に検査する。
- 外部辞書、翻訳、AIへ送る情報と、サーバー保存・削除の限界を設定画面へ明示する。

### PWA・運用

- Service Workerはアプリ本体の取得に成功した場合だけ更新し、失敗時は旧版を残す。
- HTML保存・JSONバックアップ・共有JSONは、一時リンクをDOMへ置いて保存開始後にBlob URLを破棄し、Safari系でも安定して保存する。
- manifest用PNGの署名と実寸が192px／512pxの宣言に一致することを公開前検査する。
- 個人キー付きナビゲーションURLをCache Storageのキーとして保存しない。
- 非表示タブでは15秒ポーリングを止め、前面復帰時に即時取得してから再開する。
- PWAアイコンだけでは機種変更後に復元できないことを明示し、個人リンクまたはJSONを求める。
- trashは30日、削除墓標は90日だが、期限切れの掃除は次回保存・同期時であり、
  長期休眠D1行のサーバー定期削除は未実装と明示する。

### 学習・教材

- 「この設定で出題」の例文問題は`出さない`を既定とし、`一部`または`全部`を本人が選んだ
  場合だけ有効にする。
- 前回選んだCEFR条件を他のクイズ設定と一緒に復元し、範囲ボタンと展開領域をARIAで関連付ける。
- OS共有に表示される単語帳名をWordBankへ統一する一方、旧保存キーとJSONファイル名は互換性のため維持する。
- CEFR、品詞、学習カウンタ、時刻を許可値・有限な非負値へ正規化する。
- 内蔵300語の件数、欠損、単語重複、意味重複を公開前に検査する。
- review-eventとlexical-shadowは影スキーマだけを追加し、本番の学習履歴収集や語義置換には
  接続していない。語義データには出典と確認済みライセンス状態を必須とした。

### 公開前検査

- ルートHTMLと公開HTMLの完全一致、JavaScript構文、manifest、アイコン、Service Worker、
  D1スキーマ、API契約、配布説明を検査する。
- 静的DOMのID一意性、label・ARIA参照、button型、画像alt、`target=_blank`保護を検査する。
- 公開版HTMLとローカルHTMLのSHA-256一致、公開APIのGET／405／CORS拒否、公開画面の
  同期・設定・通常クイズを確認した。

## 残っている問題と安全な対策順

| 優先 | 未解決事項 | 今すぐ変更しない理由 | 次の安全な対策 |
|---|---|---|---|
| 高 | サーバーは最新1世代だけ | 誤削除・破損後の遠隔復旧が弱い | D1容量上限を決め、5世代＋日次7日を別テーブルで影記録して復元演習 |
| 高 | 個人リンクを知る人は読書き可能 | 認証変更は既存利用者を締め出す | 認証V2へコピー移行し、旧領域を残したまま新リンク確認後に切替 |
| 高 | 長期休眠D1行にサーバーTTLがない | 誤削除と復旧猶予の設計が先 | 認証V2後、最終アクセス・復旧猶予・削除ジョブの監査ログを追加 |
| 中 | ブラウザE2EがCIで自動実行されない | GitHubトークンにworkflow更新権限がなくpush不可 | workflow権限を付け、`npm test`と主要画面smokeだけを必須化 |
| 中 | 単一HTMLが約1.6万行 | 一括分割は回帰範囲が大きい | 純粋関数とテストから段階抽出し、生成物の完全一致を保つ |
| 中 | 現行SRSはFSRSではない | 即時切替で復習日が急変する | 端末内で影計算し、30日保持率とレビュー回数を比較してから段階展開 |
| 中 | 語義・CEFR/JACET/WordNetが本番未接続 | 権利・版・語義対応の誤りが教材を汚す | provenance確認済みの影データを少量作り、人手監査後に表示のみ試す |
| 中 | 学習イベント分析が未接続 | 同意・保持・削除なしの収集は避ける | 端末内既定、明示同意、最小項目、保持期限、全削除連動を先に実装 |
| 低 | AIキー経路の実キーE2Eが未実施 | テスト用秘密を保存・共有しないため | 所有者が一時キーで数語だけ確認し、キーをログへ残さず失効する |

## リリース判断

今回の変更は既存データ形式を置換せず、追加防御、説明修正、回帰検査、未接続スキーマに限定した。
自動検査と公開smokeが成功しているため、無料・小規模運用を継続してよい。
上表の高優先3件は、移行と復旧を同時に設計できるまでは、現状を明示して保留する方が安全である。
